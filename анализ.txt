1. АРХИТЕКТУРА И ТЕХНОЛОГИИ
Выполнено:

Разработано полноценное веб-приложение на Flask с двумя интерфейсами

Использована легковесная архитектура: Python + Flask + SQLite + Jinja2

Реализована система маршрутизации

2. РЕАЛИЗАЦИЯ ДВУХ ИНТЕРФЕЙСОВ (АДМИНИСТРАТОР/ПОЛЬЗОВАТЕЛЬ)
Выполнено:

Создана таблица users с полем role (admin/user)

Разделены меню и доступные разделы в зависимости от роли

Администратор имеет доступ к: управлению книгами, арендами, напоминаниям

Пользователь имеет доступ к: каталогу книг, моим арендам, уведомлениям

3. ФУНКЦИОНАЛ АРЕНДЫ КНИГ
Выполнено:

Аренда на 3 срока: 2 недели, 1 месяц, 3 месяца

Автоматический расчет даты окончания

Блокировка книги при аренде 

Возврат книги через администратора

Визуальная индикация просроченных аренд (красный фон)

4. АВТОМАТИЧЕСКИЕ НАПОМИНАНИЯ
Выполнено:

Автоматическая проверка аренд при входе пользователя

Отправка напоминаний за 3, 2, 1 день, в день окончания и при просрочке

Сохранение истории всех напоминаний

Отображение напоминаний через flash-сообщения и в разделе "Уведомления"

5. ФИЛЬТРАЦИЯ И СОРТИРОВКА
Выполнено для пользователя (каталог книг):

Фильтры: по категории, автору, году

Сортировка: по названию, автору, категории, году, цене (8 вариантов)

Выполнено для администратора (управление книгами):

Фильтры: по категории, автору, году, доступности

Сортировка: 10 вариантов (включая по цене аренды)

Выполнено для администратора (аренды):

Фильтр по статусу (активные/возвращенные)

Сортировка: по дате окончания, начала, названию, пользователю

Выполнено для пользователя (мои аренды):

Сортировка: 6 вариантов

Фильтр по статусу

6. БЕЗОПАСНОСТЬ И ЗАЩИТА ДАННЫХ
Реализовано:

Проверка роли пользователя на каждом маршруте

Защита от редактирования арендованной книги (чекбокс блокируется)

Серверная валидация при изменении доступности книги
